HTTPS

Informacion sobre HTTPS

  • ¿Que es HTTPS?
  • ¿Por que lo debo tener?
  • 1) ¿Sobre que protocolo binario esta montado el protocolo HTTP?
  • El protocolo binario sobre el que esta montado el protocolo HTTP es TCP.
  • 2) ¿Cuales son los clientes HTTP y los servidores HTTP mas utilizados?
  • Clientes: Google Chrome, Mozilla Firefox, Safari, Microsoft Edge.
    Servidores: Apache HTTP server, Microsoft Internet Information Services, Ngnix.
  • 3) ¿Que verbos admite un commando en el requerimiento HTTP?
  • GET, POST, PUT, DELETE, HEAD, PATCH, OPTIONS, TRACE, CONNECT.
  • 4) ¿Que contenido lleva el body de un requerimiento HTTP?
  • Puede llevar texto, archivos o flujo de datos.
  • 5) ¿Que diferencia existe entre un URL y un URI?
  • URI es una cadena de caracteres que identifica un recurso de manera única. Puede ser una dirección web, un archivo, un servicio o cualquier otra cosa que pueda tener un identificador único en el contexto en el que se utiliza. URL proporciona la ubicación especifica de un recurso en una red, generalmente en Internet.
  • 6) ¿En que variables quedan almacenados datos de las respuestas HTTP?
  • Dependen del entorno o lenguaje de programación que estas usando para realizar la solicitud HTTP. Python: al realizar una solicitud HTTP con bibliotecas romo requests, los datos de la respuesta se almacenan en una variable que contiene el objeto de respuesta devuelto por la biblioteca. Java: al realizar una solicitud HTTP con bibliotecas como HttpURLConnection, podes obtener el contenido de la respuesta utilizando métodos
  • 7) ¿Que significa Virtual hosting?
  • Es una técnica utilizada en servidores web para alojar múltiples sitios web en una misma maquina física.
  • 8) ¿Por que HTTP se considera STATE LESS?
  • Se considera sin estado porque cada solicitud y respuesta HTTP se procesa de manera independiente y no se mantiene información sobre el estado de la comunicación entre el cliente y el servidor.
  • 9) ¿Como va a mejorar la velocidad de la WEB en poco tiempo?
  • HTTP3 utiliza el protocolo de transporte QUIC, que esta basado en UDP brindando una mayor velocidad de transferencia de datos y tiempo de carga mas rápidos para las paginas web.
  • 10) ¿Como va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?
  • Priorización de contenido: HTTP3 permite establecer prioridades para el contenido a transferir, lo que significa que elementos importantes para la visualización de la pagina pueden ser enviados antes que los elementos secundarios o menos críticos. Esto mejora la experiencia del usuario en ambientes de débil conexión
